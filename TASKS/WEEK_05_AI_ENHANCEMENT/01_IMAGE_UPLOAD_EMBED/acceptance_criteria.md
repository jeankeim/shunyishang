# Week 5 - Task 01: 图片上传与Embedding - 验收标准

## 功能验收标准

### 1. 图片上传功能
| 验收项 | 验收标准 | 测试方法 |
|--------|----------|----------|
| 格式支持 | 支持 JPG、PNG 格式 | 上传不同格式图片验证 |

| 拖拽上传 | 支持拖拽文件到上传区域 | 手动拖拽测试 |
| 点击上传 | 支持点击选择文件 | 手动点击测试 |
| 预览功能 | 上传后显示图片预览 | 视觉验证 |
| 进度显示 | 大文件显示上传进度 | 上传大文件观察 |

### 2. 图片 Embedding 生成
| 验收项 | 验收标准 | 测试方法 |
|--------|----------|----------|
| 向量维度 | 生成 512 维向量 | 检查返回数据 |
| 生成时间 | < 2秒 | 计时测试 |
| 模型加载 | 单例模式，仅加载一次 | 查看日志 |
| 错误处理 | 无效图片返回友好错误 | 上传损坏图片测试 |

### 3. 以图搜图功能
| 验收项 | 验收标准 | 测试方法 |
|--------|----------|----------|

| 响应时间 | < 500ms | 计时测试 |

### 4. 图片存储
| 验收项 | 验收标准 | 测试方法 |
|--------|----------|----------|
| 文件命名 | 使用 UUID 命名，无冲突 | 上传同名文件 |

## 接口验收标准

### 1. 上传图片接口
```http
POST /api/v1/image/upload
Content-Type: multipart/form-data

Request:
  file: File (required)

Response (200):
{
  "image_id": "uuid",
  "url": "/static/images/uuid.jpg",
  "embedding": [512维向量],
  "width": 800,
  "height": 600
}

Response (400):
{
  "detail": "不支持的文件格式，仅支持 JPG/PNG"
}
```

### 2. 以图搜图接口
```http
POST /api/v1/image/search
Content-Type: application/json

Request:
{
  "image_id": "uuid",
  "top_k": 10
}

Response (200):
{
  "results": [
    {
      "item_id": "uuid",
      "name": "衣物名称",
      "similarity": 0.92,
      "image_url": "/static/images/..."
    }
  ]
}
```
