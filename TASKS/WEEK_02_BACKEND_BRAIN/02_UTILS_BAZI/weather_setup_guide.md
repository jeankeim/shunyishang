# 天气功能配置指南

## 功能概述

本项目实现了完整的浏览器自动定位和真实天气获取功能，包括：

1. **浏览器自动定位** - 使用HTML5 Geolocation API
2. **真实天气数据** - 集成和风天气API
3. **智能逆地理编码** - 高德地图API + 本地算法双备份
4. **定位历史记录** - 自动保存用户常用位置
5. **完善的错误处理** - 用户友好的错误提示

## 配置步骤

### 1. 和风天气API配置

#### 注册账号
1. 访问 [和风天气开发者平台](https://dev.qweather.com/)
2. 注册免费账号
3. 创建应用获取API Key

#### 配置环境变量
```bash
# 在 .env 文件中添加
WEATHER_API_KEY=你的和风天气API密钥
```

### 2. 高德地图API配置（可选但推荐）

#### 注册账号
1. 访问 [高德开放平台](https://lbs.amap.com/)
2. 注册账号并创建应用
3. 申请"Web服务"API权限
4. 获取API Key

#### 配置环境变量
```bash
# 在 .env 文件中添加
AMAP_API_KEY=你的高德地图API密钥
```

#### 前端配置
在 `apps/web/.env.local` 文件中添加：
```bash
NEXT_PUBLIC_AMAP_API_KEY=你的高德地图API密钥
```

## 功能特性

### 📍 浏览器自动定位
- **高精度定位** - 使用enableHighAccuracy选项
- **智能缓存** - 5分钟内重复定位使用缓存
- **超时保护** - 15秒超时限制
- **权限引导** - 详细的权限拒绝指引

### 🌤️ 真实天气数据
- **实时天气** - 温度、湿度、风力等详细信息
- **五行映射** - 自动将天气映射到对应的五行属性
- **智能缓存** - 天气数据缓存15分钟减少API调用
- **降级机制** - 无API Key时自动切换到模拟数据

### 🏙️ 智能逆地理编码
1. **首选方案** - 高德地图API（精确）
2. **备选方案** - 本地坐标匹配算法
3. **兜底方案** - 最近距离城市计算

### 📚 定位历史
- 自动保存最近5个定位城市
- 智能去重
- 本地存储（localStorage）

## 错误处理机制

### 定位错误分类
- `PERMISSION_DENIED` - 权限被拒绝
- `POSITION_UNAVAILABLE` - 位置不可用
- `TIMEOUT` - 定位超时
- `CITY_NOT_FOUND` - 无法识别城市
- `UNSUPPORTED` - 浏览器不支持

### 用户体验优化
- 清晰的错误提示信息
- 自动清除临时错误消息
- 手动选择城市作为备选方案
- 重试机制引导

## 注意事项

1. **API调用限制** - 免费版API有调用次数限制
2. **HTTPS要求** - 生产环境需使用HTTPS才能使用定位功能
3. **移动端适配** - 移动设备定位精度更高
4. **隐私保护** - 仅在用户明确授权时获取位置信息

## 故障排除

### 常见问题

1. **定位按钮无响应**
   - 检查浏览器是否支持Geolocation
   - 确认HTTPS环境（生产环境）

2. **无法获取真实天气**
   - 检查WEATHER_API_KEY是否正确配置
   - 查看后端日志确认API调用状态

3. **城市识别不准确**
   - 配置高德地图API KEY提高精度
   - 检查网络连接是否正常

4. **定位超时**
   - 增加timeout参数值
   - 检查GPS信号强度
